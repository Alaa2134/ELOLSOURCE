# 🧾 نظام كاشير السقا للأدوات المنزلية

برنامج كاشير ونقطة بيع متكامل — واجهة عربية بالكامل، شغال أونلاين وأوفلاين، بيترفع على Vercel، مع تخزين سحابي وإرسال فواتير ورسائل شكر واتساب.

## ✨ المميزات

- 🧾 **شاشة فاتورة بيع** بنفس شكل الشاشة القديمة (شريط برتقالي، جدول أصناف) لكن أسرع وأحدث
- 🖨️ **طباعة "بيان أسعار"** A4 مطابق لشكل فاتورة السقا الحالية بالأرقام العربية (١٢٣)
- 📦 **إدارة أصناف ومخزون**: خصم تلقائي مع كل بيع، تنبيه نقص المخزون، استيراد بالنسخ من إكسل، تصدير CSV
- 🔫 **دعم ماسح الباركود** — امسح الكود وهيضيف الصنف فوراً
- 👥 **إدارة عملاء** مع حساب المديونيات (البيع الآجل) وإجمالي التعامل
- 📈 **تقارير**: مبيعات باليوم، الأصناف الأكثر مبيعاً، ربح تقريبي، طباعة التقرير
- ☁️ **تخزين سحابي (Supabase)**: مزامنة تلقائية بين الأجهزة + شغل كامل بدون إنترنت والمزامنة لما النت يرجع
- 💬 **واتساب**: رسالة شكر تلقائية بعد كل فاتورة + رابط الفاتورة الإلكترونية للعميل
- 📱 **QR على الفاتورة المطبوعة** — العميل يمسحه ويشوف فاتورته على الموقع
- 💾 **نسخ احتياطي** بضغطة زر (تحميل واسترجاع)
- 🔒 دخول برقم سري

## 🚀 التشغيل محلياً

```bash
npm install
npm run dev
```

افتح http://localhost:3000 — الرقم السري الافتراضي: **1234** (غيّره من الإعدادات).

## ▲ الرفع على Vercel

1. ادخل على [vercel.com](https://vercel.com) وسجّل بحساب GitHub
2. Add New → Project → اختر الريبو ده → Deploy (مفيش أي إعدادات إضافية مطلوبة)
3. بعد الرفع، حط رابط الموقع (مثل `https://saqqa.vercel.app`) في **الإعدادات → رابط الموقع** عشان روابط الفواتير المرسلة للعملاء تشتغل

## ☁️ تفعيل التخزين السحابي (Supabase — مجاني)

1. اعمل حساب على [supabase.com](https://supabase.com) → New Project
2. من **SQL Editor** شغّل محتوى الملف `supabase/schema.sql`
3. من **Project Settings → API** انسخ `URL` و `anon key`
4. في Vercel: **Settings → Environment Variables** أضف:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
5. اعمل Redeploy — خلاص، كل البيانات هتتزامن تلقائياً بين كل الأجهزة

> بدون Supabase البرنامج شغال عادي بتخزين محلي على الجهاز، بس رابط الفاتورة اللي بيتبعت للعميل مش هيفتح من موبايله إلا مع السحابة.

## 💬 بوابة الواتساب (تسجيل دخول QR)

البوابة خدمة صغيرة بتشتغل على أي جهاز شغال طول الوقت (كمبيوتر المحل / سيرفر رخيص / Raspberry Pi).
**ليه مش على Vercel؟** لأن Vercel serverless مبيسمحش باتصال واتساب مستمر.

```bash
cd whatsapp-gateway
cp .env.example .env      # عدّل WA_TOKEN لرمز سري خاص بك
npm install
npm start
```

بعدين في البرنامج: صفحة **واتساب** → حط رابط البوابة (مثل `http://localhost:3900`) والرمز → هيظهر QR → امسحه من واتساب (الأجهزة المرتبطة → ربط جهاز).

> لو البوابة على جهاز المحل والموقع على Vercel، هتحتاج توصلهم — أسهل حل: شغّل البوابة على سيرفر VPS رخيص، أو استخدم نفق زي Cloudflare Tunnel.

### 🛡️ نظام الحماية من الحظر (مهم جداً)

الحظر بييجي من سلوك يشبه السبام. البوابة بتطبق تلقائياً:

| الحماية | الافتراضي | ليه |
|---|---|---|
| تأخير عشوائي بين الرسائل | 25–70 ثانية | محاكاة إرسال يدوي، مش روبوت |
| حد أقصى يومي | 150 رسالة | الأرقام الجديدة بتتحظر من الكم الكبير |
| ساعات إرسال | 9ص – 10م | الإرسال بالليل سلوك مشبوه |
| فحص الرقم قبل الإرسال | دائماً | الإرسال لأرقام مش على واتساب أسرع طريق للحظر |
| "يكتب الآن..." قبل الرسالة | دائماً | سلوك بشري طبيعي |

**ونصائح لازم تلتزم بيها:**

1. استخدم **رقم مخصص للشغل** مش رقمك الشخصي
2. **سخّن الرقم**: أول أسبوع ابعت قليل (10–20 رسالة/يوم) وزوّد تدريجياً — عدّل `DAILY_CAP` في `.env`
3. ابعت **للعملاء اللي اشتروا فعلاً بس** (رسالة الفاتورة معاملة مشروعة، مش إعلان)
4. متبعتش **نفس النص بالظبط** لمئات الأشخاص — الرسالة فيها اسم العميل ورقم فاتورته فده طبيعي
5. لو عميل قال "بطّل تبعتلي" — شيل رقمه فوراً (البلاغات هي سبب الحظر الأول)
6. **البديل الآمن 100%**: زر 💬 الأخضر في البرنامج بيفتح واتساب العادي برسالة جاهزة (wa.me) — مفيش أي خطر منه إطلاقاً
7. **للمدى الطويل والأمان الكامل**: فكّر في WhatsApp Business Cloud API الرسمي من Meta (مجاني لحد 1000 محادثة/شهر) — ده مستحيل يتحظر لأنه رسمي

## 🗂️ هيكل المشروع

```
app/            صفحات البرنامج (Next.js App Router)
  pos/          شاشة فاتورة البيع
  print/[id]/   طباعة بيان الأسعار A4
  inv/[id]/     صفحة الفاتورة العامة (للعميل من رابط الواتساب / QR)
  ...
components/     مكونات مشتركة (القائمة الجانبية، نموذج الفاتورة)
lib/            طبقة البيانات والمزامنة + الواتساب + التنسيقات
supabase/       سكيما قاعدة البيانات السحابية
whatsapp-gateway/  بوابة الواتساب (Baileys + حماية من الحظر)
```
